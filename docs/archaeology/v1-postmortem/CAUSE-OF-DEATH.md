## Counterfactuals

1. **If each concept lived in one crate until cross-crate reuse forced a split, the workspace would have ~22 crates instead of 62.** Evidence: The 28 triad crates carry 1 concept per 2.3 crates. `methodology-{domain,contracts,adapters}` = 1,128 lines total across 3 Cargo.tomls — a single 1,100-line module would compile in 1/3 the time and never need an ACL.

2. **If production binaries had been wired to real infrastructure on day 1, InMemory doubles could not have leaked into production.** Evidence: `coder-daemon.rs` has 4 InMemory references and 1 redis reference today. `commands-daemon.rs` has 5 InMemory, 0 redis. `coder-mcp-server.rs` has 2 InMemory, 0 redis. These are the binaries the dashboard was supposed to prove — they never got wired, and #430 (P0 stream-name mismatch) survived for weeks because no live path exercised them.

3. **If the methodology had stayed as skills (team topology) instead of being coded into `devkit-gates` + `quality-contracts/judges/` + `agency-ci-generator`, removing a gate would be a config change instead of a code release.** Evidence: 30,556 LOC (24% of the codebase) is methodology-enforcement machinery — `devkit-gates` alone is larger than the 14 daemons combined. Every change to the methodology required a Rust PR; every Rust PR required the methodology. Circular dependency between the runtime and its own rules is why commit cadence fell off a cliff in March.
